Q2.
ANS:
i. onCreate called when activity is first created.
ii.onStart called when activity is becoming visible to the user
iii.onResume called when activity will start interacting with the user
iv.onPause called when activity is not visible to the user. Therefore the activity will be paused.
v.onStop called when activity is no longer visible to the user. And the data will be stored at the background operation.
vi. onDestory called before the activity is destroyed and all the stored data will be destroyed.

A coroutine is a concurrency design pattern that you can use on Android to simplify code that executes asynchronously. 

On Android, coroutines help to manage long-running tasks that might otherwise block the main thread and cause your app to become unresponsive. Over 50% of professional developers who use coroutines have reported seeing increased productivity.
